# Руководство пользователя

## 1. Что это за программа

Это учебная мини-СУБД на C++. Она работает через консоль: пользователь вводит SQL-запросы, программа выполняет их и сохраняет данные на диск.

Данные не пропадают после закрытия программы. Таблицы и записи хранятся в папке `data/databases/`.

## 2. Сборка проекта

Откройте терминал в корне проекта и выполните:

```bash
./build.sh
```

Скрипт:

- определит операционную систему;
- проверит наличие CMake и make;
- при необходимости попробует установить зависимости через `apt`, `brew` или `pacman`;
- соберет проект в папку `build/`.

После успешной сборки появятся:

```text
build/libdb_mvp.a
build/db_mvp
```

`libdb_mvp.a` - C++ библиотека СУБД.  
`db_mvp` - консольное приложение для запуска СУБД.

## 3. Запуск

Запуск выполняется командой:

```bash
./run.sh localhost 7432
```

Где:

- `localhost` - host;
- `7432` - port.

В текущем MVP настоящего сетевого сервера нет. Эти параметры принимаются и выводятся в интерфейсе как параметры запуска.

Если исполняемый файл еще не собран, `run.sh` автоматически запустит `build.sh`.

## 4. Выход из программы

Для выхода введите:

```text
exit
```

или:

```text
quit
```

## 5. Как вводить SQL-запросы

Каждый SQL-запрос должен заканчиваться точкой с запятой:

```sql
SELECT * FROM users;
```

Можно вводить запрос в одну строку или в несколько строк. Программа выполнит запрос только после символа `;`.

## 6. Создание базы данных

```sql
CREATE DATABASE demo;
```

После создания база `demo` автоматически становится текущей.

Чтобы выбрать уже существующую базу данных, используйте:

```sql
USE demo;
```

`USE` - дополнительная удобная команда этого проекта.

## 7. Удаление базы данных

```sql
DROP DATABASE demo;
```

Команда удаляет базу данных вместе с ее таблицами и файлами хранения.

## 8. Создание таблицы

```sql
CREATE TABLE users (
    id INT,
    name VARCHAR(50),
    age INT,
    rating FLOAT,
    active BOOL
);
```

Поддерживаемые типы данных:

```text
INT
FLOAT
BOOL
TEXT
VARCHAR(n)
```

## 9. Удаление таблицы

```sql
DROP TABLE users;
```

## 10. Вставка данных

Вставка одной строки:

```sql
INSERT INTO users VALUES (1, 'Alice', 20, 4.5, TRUE);
```

Вставка нескольких строк:

```sql
INSERT INTO users VALUES
    (1, 'Alice', 20, 4.5, TRUE),
    (2, 'Bob', 25, 3.7, FALSE),
    (3, 'Carol', 30, 4.9, TRUE);
```

Вставка с явным указанием столбцов:

```sql
INSERT INTO users (id, name, active) VALUES
    (4, 'Dave', TRUE);
```

Если часть столбцов не указана, в них будет записано значение `NULL`.

## 11. Выборка данных

Вывести все столбцы:

```sql
SELECT * FROM users;
```

Вывести только выбранные столбцы:

```sql
SELECT name, age FROM users;
```

Выборка с условием:

```sql
SELECT name, age FROM users WHERE active = TRUE;
```

## 12. Условия WHERE

Поддерживаются операторы сравнения:

```text
=
!=
<
>
<=
>=
```

Поддерживаются логические операторы:

```text
AND
OR
NOT
```

Также поддерживаются скобки:

```sql
SELECT * FROM users
WHERE active = TRUE AND (age >= 18 OR rating > 4.8);
```

Пример с `NOT`:

```sql
SELECT * FROM users WHERE NOT active = FALSE;
```

## 13. Обновление данных

```sql
UPDATE users SET rating = 5.0 WHERE name = 'Alice';
```

После выполнения программа выводит количество измененных строк:

```text
Affected rows: 1
```

## 14. Удаление данных

```sql
DELETE FROM users WHERE active = FALSE;
```

После выполнения программа выводит количество удаленных строк:

```text
Affected rows: 1
```

## 15. Демо-сценарий

В проекте есть готовый файл:

```text
tests/demo.sql
```

Запустить его можно так:

```bash
./run.sh localhost 7432 < tests/demo.sql
```

Он проверяет:

- создание базы данных;
- создание таблицы;
- вставку нескольких строк;
- выборку всех строк;
- выборку с проекцией;
- выборку с `WHERE`;
- обновление с `WHERE`;
- удаление с `WHERE`.

## 16. Проверка сохранения данных

Сначала запустите демо:

```bash
./run.sh localhost 7432 < tests/demo.sql
```

Затем запустите программу заново:

```bash
./run.sh localhost 7432
```

Введите:

```sql
USE demo;
SELECT * FROM users;
```

Если таблица и записи вывелись, значит данные сохранились после перезапуска.

## 17. Формат вывода

Для `SELECT` результат выводится таблицей:

```text
+----+-------+-----+
| id | name  | age |
+----+-------+-----+
| 1  | Alice | 20  |
+----+-------+-----+
Rows: 1
```

Для `INSERT`, `UPDATE`, `DELETE`:

```text
Affected rows: N
```

Для `CREATE` и `DROP`:

```text
Таблица создана: users
База данных удалена: demo
```

При ошибке:

```text
Ошибка: описание ошибки
```

## 18. Ограничения MVP

В этой версии не реализованы:

- настоящий сетевой сервер;
- индексы;
- транзакции;
- права пользователей;
- первичные и внешние ключи;
- полный стандарт SQL.

Проект реализует учебный минимум, достаточный для демонстрации базовых принципов работы реляционной СУБД.
